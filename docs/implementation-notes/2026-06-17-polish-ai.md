# 2026-06-17 — Field-grade UX polish + differentiating AI

Two threads: a UX-polish pass (mobile, WCAG, i18n parity, status
consistency) and the differentiating intelligence — an agronomy copilot
and a photo pest/disease identifier, both reusing the existing OpenRouter
/ Claude surface.

## AI — design

**One fail-safe LLM client.** `src/app-layer/ai/llm-client.ts` wraps the
same OpenRouter endpoint + `OPENROUTER_API_KEY` / `OPENROUTER_MODEL` the
risk-assessment provider uses, but exposes a GENERIC text+vision JSON
completion (`llmCompleteJson`, `isLlmConfigured`, `imageDataUri`). It
NEVER throws — returns `null` when the key is absent or a call fails. AI
is enrichment: a spray signal still fires and a photo still uploads with
the LLM off.

**Agronomy copilot.** When `evaluateLocationSignals` claims a NEW
AgroSignal it enqueues `agronomy-copilot` (gated on `isLlmConfigured`,
enqueue failure swallowed). The job loads the signal's weather window +
crop context, asks Claude for a plain-language explanation
(`explanation` / `factors` / `whatIf` / `confidence`, Zod-validated),
merges it under `AgroSignal.detailsJson.copilot`, and fires an
`AGRO_COPILOT_READY` notification (reuses the GENERAL type + the agro
dedupe key). A small honest `computeGddSum` feeds GDD context.

**Photo pest/disease ID.** `uploadLogEntryPhoto` enqueues `photo-pest-id`
for image uploads (gated). The job reads the stored image (capped 8 MB)
→ base64 → Claude vision, conservatively identifies pest/disease/
deficiency (`identified` / `category` / `name` / `recommendation` /
`confidence`), writes it under `LogEntry.attributesJson.pestId` (new
column — migration `add_log_entry_attributes`), and notifies the
uploader. Rendered live on the journal photos tab.

**Confidence + human-review, always.** Every AI surface renders through
`<AiSuggestionCard>` (`src/components/ag/`), which is structurally
incapable of omitting the two invariants: a confidence StatusBadge (icon
+ text, never colour-only) and the "AI suggestion — verify with an
agronomist." disclaimer. `<CopilotCard>` and `<PhotoIdCard>` are thin
adapters over it. AI is triage, not diagnosis.

## Polish — design

- **Status consistency.** `src/components/ag/ag-status.tsx` consolidates
  every ag entity's `status → variant + label` map (cropPlan, season,
  planting, operationParcel, bin, contract) behind `<AgStatusBadge>` /
  `agStatusVariant`. Seven pages dropped their inline duplicate maps; a
  raw-`<span>` status on the field-operation panel became a proper
  icon+text badge. DRY, not a recolour.
- **i18n en/bg parity.** A new `ag` namespace (AI strings incl. the
  disclaimer + ag status labels) lands in BOTH `messages/en.json` and
  `messages/bg.json` with real Bulgarian translations; the
  `i18n-completeness` guardrail (en↔bg key + placeholder parity, CI-run)
  stays green.
- **WCAG 2.1 AA + responsive mobile + Playwright mobile.** jest-axe over
  the five ag pages, non-colour-only status, map keyboard/ARIA + touch
  targets, map+parcel-list stacking below 768px, and mobile device
  projects + a responsive E2E spec.

## Decisions

- **Reuse the OpenRouter surface, not the risk provider.** The existing
  provider is risk-typed; a thin generic client avoids shoehorning ag
  features into a risk shape and gives one place for the vision path.
- **Async jobs, never inline.** AI calls are slow + flaky; they run as
  on-demand BullMQ jobs so the signal/upload path stays fast and the AI
  call is retryable + isolated.
- **Stable dedupe.** Copilot keyed on the AgroSignal unique tuple, photo
  ID on `(logEntry, fileRecord)` — a retry re-generates and overwrites,
  the notification dedupe key prevents a double-notify.
- **Disclaimer is a component invariant, not a convention.** Routing all
  AI output through `AiSuggestionCard` makes "show the human-review
  disclaimer" impossible to forget.
